# Jogak

> **조각** — Korean for "piece". 조각들이 모여 하나의 조각을 만든다.

**Jogak**은 가벼운 컴포넌트 쇼케이스 라이브러리입니다. Storybook의 동일한 작업 흐름(`dev` / `build` / static deploy)을 유지하면서, 더 가벼운 스택으로 갈아 끼울 수 있는 **drop-in 대체**입니다 — 단일 Vite, 사용자 컴포넌트는 사용자 자신의 Vite scope에 마운트되는 iframe, 타입 추출은 ts-morph, Web Components는 Preact 런타임만.

> **Storybook에서 오셨나요?** [마이그레이션 가이드](/ko/docs/migration-from-storybook) · [전체 비교 + addon 매트릭스](/ko/docs/comparison)

> 현재 상태: **`1.0.0`** — 정식 릴리스. Public API는 고정, 이후 변경은 semver를 따릅니다.

## 5개 프레임워크, 하나의 뷰어

```ts
const meta = {
  title: 'Components/Button',
  component: Button,
  framework: 'vue', // 'react' | 'next' | 'vue' | 'svelte' | 'web-components'
} satisfies JogakMeta
```

동일한 `*.jogak.{ts,tsx}` 컨벤션이 **React · Next.js · Vue 3 · Svelte 5 · Web Components** 5개 프레임워크에서 동작합니다. 사용자 컴포넌트는 사용자 자신의 Vite scope에 마운트되므로 (iframe isolation) Tailwind, 디자인 시스템, 번들러 설정이 그대로 보입니다.

## 빠르게 시작하기

```bash
pnpm add -D @jogak/cli @jogak/core

# 컴포넌트 옆에 *.jogak.tsx 작성 후:
npx jogak dev          # http://localhost:5173
npx jogak build        # static 산출물 → ./jogak-static
```

→ [Quick start](/ko/docs)로 이어집니다.

## 이름의 의미

> "조각들이 모여 하나의 조각을 만든다."
>
> Storybook은 많은 팀에게 무겁고 오류가 잦습니다. Jogak은 같은 의도 — **컴포넌트 조각들을 모아두는 곳** — 을 공유하면서도, drop-in으로 갈아 끼울 만큼 작게 유지하는 것을 목표로 합니다.
>
> 잦은 오류와 무거운 Storybook과 목표하는 바와 지향점이 비슷합니다. 사용해 보시고 많은 피드백 부탁드립니다.
