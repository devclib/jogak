# Jogak

> **조각** — Korean for "piece". 조각들이 모여 하나의 조각을 만든다.

## 더 가벼운 Storybook 대체 — **1.0 정식 릴리스**

- **한 줄 install, 몇 초 만에 실행.** `pnpm add -D @jogak/cli @jogak/core` → `jogak dev` → 끝. wizard, `.storybook/` scaffold, framework preset 없음. Zero-config 동작.
- **번들 7배 작음. 콜드 스타트 3초 미만.** 단일 Vite 인스턴스, lazy 가상 모듈, `child_process` 격리된 ts-morph. Idle RSS가 카탈로그 크기에 거의 비례하지 않습니다.
- **5개 framework, 하나의 컨벤션.** React · Next.js · Vue 3 · Svelte 5 · Web Components를 같은 `*.jogak.{ts,tsx}` schema로.
- **당신의 Vite / Tailwind / plugin 그대로.** 사용자 컴포넌트는 사용자 자신의 Vite scope에 iframe으로 mount — 번들러가 대체되지 않습니다.
- **Public API 고정.** 1.0.0 정식 — 이후 semver.

**Storybook에서 오셨나요?** → [마이그레이션 가이드 (5단계)](/ko/docs/migration-from-storybook) · [비교 + addon 매트릭스](/ko/docs/comparison)

## 5분 Quick start

```bash
# 1. 설치
pnpm add -D @jogak/cli @jogak/core

# 2. jogak.config.ts + Button 예시를 한 방에 scaffold
npx jogak init

# 3. 실행
npx jogak dev          # http://localhost:5173
npx jogak build        # static 산출물 → ./jogak-static
```

`jogak init`은 idempotent — 재실행해도 안전. `--force`로 덮어쓰기 가능. 수동 entry 작성은 [Quick start guide](/ko/docs) 참조.

`jogak-static/`을 어디든 배포: GitHub Pages, Vercel, Netlify, S3.
→ framework peer + 옵션은 [Quick start guide](/ko/docs)로.

## 5개 framework, 하나의 컨벤션

```ts
const meta = {
  title: 'Components/Button',
  component: Button,
  framework: 'vue', // 'react' | 'next' | 'vue' | 'svelte' | 'web-components'
} satisfies JogakMeta
```

동일한 `*.jogak.{ts,tsx}` 파일이 **React · Next.js · Vue 3 · Svelte 5 · Web Components**에서 동작합니다. Prop 타입은 ts-morph로 자동 추출 — Controls, Actions, Source view가 설정이 아닌 생성됩니다.

## 기본 built-in 기능

Storybook essential addon 대응 — addon API 설정 불필요:

- **Controls** — TypeScript 타입에서 자동
- **Actions** — 함수 prop 자동 spy
- **Viewport** — Mobile / Tablet / Desktop
- **Backgrounds** — white / dark / transparent
- **Source view** — 현재 args에서 사용 snippet
- **A11y** — axe-core 자동 스캔 (`axe-core` 설치 시 활성)

built-in / planned / out of scope 전체는 [addon 매트릭스](/ko/docs/comparison#addon-매트릭스)를 참조.

## 이름의 의미

> "조각들이 모여 하나의 조각을 만든다."
>
> Storybook은 많은 팀에게 무겁고 오류가 잦습니다. Jogak은 같은 의도 — **컴포넌트 조각들을 모아두는 곳** — 을 공유하면서도, drop-in으로 갈아 끼울 만큼 작게 유지하는 것을 목표로 합니다.
