# Next.js

Next.js 14 / 15 (App Router와 Pages Router 모두) 를 **next** 빌더 어댑터로 지원합니다. RSC + Client Component가 나란히 마운트됩니다.

## 셋업

```bash
pnpm add -D @jogak/cli @jogak/core
```

PostCSS로 Tailwind v4:

```bash
pnpm add -D @tailwindcss/postcss
```

```js filename="postcss.config.mjs"
export default { plugins: { '@tailwindcss/postcss': {} } }
```

```ts filename="jogak.config.ts"
import { defineJogakConfig } from '@jogak/core'

export default defineJogakConfig({
  framework: 'next',
  globalCss: 'src/app/globals.css',
})
```

## App Router

Server Component:

```tsx filename="src/components/ui/Badge.tsx"
// 'use client' 없음 — 순수 Server Component
export function Badge({ tone = 'info', children }: { tone?: 'info' | 'success'; children: string }) {
  return <span className={`badge badge-${tone}`}>{children}</span>
}
```

Client Component:

```tsx filename="src/components/ui/Button.tsx"
'use client'
import { useState } from 'react'

export function Button({ label }: { label: string }) {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount((c) => c + 1)}>{label} ({count})</button>
}
```

둘 다 `meta.component`로 동작합니다 — Jogak의 next renderer가 올바르게 라우팅합니다. Server Component는 server tree가 마운트되고, Client Component는 state와 함께 hydrate됩니다.

## 호스트 임베딩 (Next.js App Router 페이지)

`jogak dev`를 건너뛰고 기존 Next.js 페이지에 카탈로그를 임베딩:

```bash
pnpm jogak generate --patterns 'src/**/*.jogak.tsx'
# → .jogak/registry.ts
```

```tsx filename="src/app/jogak/page.tsx"
'use client'
import dynamic from 'next/dynamic'
import { entries } from '../../../.jogak/registry'

const JogakApp = dynamic(() => import('@jogak/ui').then((m) => ({ default: m.JogakApp })), {
  ssr: false,
})

export default function Page() {
  return <JogakApp entries={entries} codeTheme="vsDark" />
}
```

```js filename="next.config.mjs"
export default {
  transpilePackages: ['@jogak/core', '@jogak/ui'],
  webpack(config) {
    config.resolve.extensionAlias = { '.js': ['.ts', '.tsx', '.js'] }
    return config
  },
}
```
