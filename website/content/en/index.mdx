# Jogak

> Pronounced "**조각**" — Korean for **"piece"**. Small pieces gathered into a whole.

## A leaner Storybook alternative — **stable at 1.0**

- **Install in one line, run in seconds.** `pnpm add -D @jogak/cli @jogak/core` → `jogak dev` → done. No wizard, no `.storybook/` scaffold, no framework preset. Zero-config works.
- **4× smaller bundle. Under 2s cold start.** Single Vite instance, lazy virtual modules, `child_process`-isolated ts-morph. Idle RSS barely scales with catalog size.
- **5 frameworks, one convention.** React · Next.js · Vue 3 · Svelte 5 · Web Components with the same `*.jogak.{ts,tsx}` schema.
- **Your Vite / Tailwind / plugins stay yours.** User components mount in your own Vite scope through an iframe — your bundler is not replaced.
- **Public API frozen.** 1.0.0 stable — semver from here on.

**Coming from Storybook?** → [Migration guide (5 steps)](/en/docs/migration-from-storybook) · [Comparison + addon matrix](/en/docs/comparison)

## Quick start (5 minutes)

```bash
# 1. Install
pnpm add -D @jogak/cli @jogak/core

# 2. Scaffold jogak.config.ts + Button example in one command
npx jogak init

# 3. Run
npx jogak dev          # http://localhost:5173
npx jogak build        # static output → ./jogak-static
```

`jogak init` is idempotent — safe to re-run. Use `--force` to overwrite. See [Quick start guide](/en/docs) for hand-writing the entry instead.

Deploy `jogak-static/` anywhere: GitHub Pages, Vercel, Netlify, S3.
→ Continue to the [Quick start guide](/en/docs) for framework peers and options.

## 5 frameworks, one convention

```ts
const meta = {
  title: 'Components/Button',
  component: Button,
  framework: 'vue', // 'react' | 'next' | 'vue' | 'svelte' | 'web-components'
} satisfies JogakMeta
```

The same `*.jogak.{ts,tsx}` file works across **React · Next.js · Vue 3 · Svelte 5 · Web Components**. Prop types are auto-extracted (via ts-morph) — Controls, Actions, and Source view are generated, not configured.

## What you get in the box

Storybook addon parity for the essentials — no addon API to configure:

- **Controls** — auto from TypeScript types
- **Actions** — function props auto-spied
- **Viewport** — Mobile / Tablet / Desktop
- **Backgrounds** — white / dark / transparent
- **Source view** — usage snippet from current args
- **A11y** — axe-core auto-scan (install `axe-core` to enable)

See the full [addon matrix](/en/docs/comparison#addon-matrix) for what's built in, planned, and out of scope.

## Why "Jogak"?

> "조각들이 모여 하나의 조각을 만든다" — _Small pieces come together to form one whole._
>
> Storybook is heavy and error-prone for many teams. Jogak shares the same intent — a place to **gather component pieces** — but stays small enough to ship as a drop-in.
