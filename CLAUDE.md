# Jogak

스토리북을 대체하는 디자인시스템 쇼케이스 라이브러리.  
React SPA / Next.js SSR+Server Components / Vanilla JS(Web Components) 지원.  
기술 스택: TypeScript + React + Vite + pnpm workspaces.

## 작업 원칙 (Karpathy 4원칙)

사용자의 **모든 요청을 처리할 때** 다음 4원칙을 적용한다. 질문 응답·코드 작성·디버깅·설계 제안 등 작업 종류를 가리지 않는다.

1. **Think Before Coding** — 코딩 전에 멈추고 생각한다. 가정하지 않고, 불명확한 부분은 먼저 질문하며, 트레이드오프를 드러낸다. "검색을 빠르게" 같은 모호한 요청은 해석 옵션(응답 시간 / 처리량 / 체감 속도 등)을 제시하고 확인받은 뒤 진행한다.
2. **Simplicity First** — 요청되지 않은 기능과 과도한 추상화를 피한다. 100줄이면 될 일을 1000줄 구조로 만들지 않는다. 전략 패턴·추상 클래스 등은 실제로 필요해진 뒤에 도입한다.
3. **Surgical Changes** — 변경한 모든 라인이 요청과 1:1로 추적 가능해야 한다. 요청 범위 밖의 따옴표 스타일·타입 힌트·독스트링·리팩터링을 함께 바꾸지 않는다. 기존 코드 스타일을 유지한 채 요청한 부분만 수정한다.
4. **Goal-Driven Execution** — 검증 가능한 성공 기준을 세우고 달성될 때까지 반복한다. "버그 수정"이 아니라 "버그 재현 테스트를 작성한 뒤 통과시키기"처럼 완료 시점을 객관적으로 판정할 수 있는 목표를 설정한다.
